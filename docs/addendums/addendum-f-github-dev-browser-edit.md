# Optional: Edit Files in the Browser with github.dev

Use this addendum if you want to make a quick edit to your repo from the browser without opening GitHub Desktop or an AI coding tool. It is optional. The required tutorial path uses GitHub Desktop and an AI tool instead.

## When this is useful

- You spotted a typo in a file and want to fix it from any computer without installing anything.
- You are not at your usual computer and just need a quick text edit.
- You want to see what GitHub's built-in browser editor looks like.

## Instructions

### Part A: Open the web editor

1. On GitHub, open your `github-startup` repository.
2. Press the **`.`** (period / dot) key on your keyboard.
3. A VS Code-like editor opens in your browser tab. This is **github.dev**. No install needed.

### Part B: Make a change

1. In the file list on the left, click a file to open it (try `index.html`).
2. Edit something simple, like a heading or your name.
3. You should see a blue dot on the file tab showing it has unsaved changes.

### Part C: Commit your change

1. Click the **branch icon** on the left sidebar (3rd from top, looks like a fork in a road).
2. You will see your changed file listed under **Changes**.
3. In the text box that says "Message", type something like `Fix typo in heading`.
4. Click the blue **Commit & Push** button.

That is it. Your change is saved to GitHub and your live GitHub Pages site will update within a few minutes.

## Tool note

`github.dev` is fine for small edits but cannot read your local files or run an AI coding tool. For building or restructuring your dashboard, use the main tutorial path (Desktop + AI tool).
